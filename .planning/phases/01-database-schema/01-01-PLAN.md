```typescript
export const monitorType = pgEnum('monitor_type', ['http', 'ping'])
export const checkStatus = pgEnum('check_status', ['up', 'down', 'pending'])
```

**monitors table (per D-04, D-05, D-06):**
Columns:
- `id` — uuid primary key, defaultRandom()
- `name` — text, not null
- `type` — monitorType enum, not null
- `url` — text, nullable (HTTP monitors only)
- `host` — text, nullable (ping monitors only)
- `interval` — integer (seconds), not null, default 60
- `enabled` — boolean, not null, default true
- `showOnStatusPage` — boolean (column name: `show_on_status_page`), not null, default true  ← per D-06
- `currentStatus` — checkStatus enum (column: `current_status`), not null, default 'pending'  ← per D-04
- `consecutiveFailures` — integer (column: `consecutive_failures`), not null, default 0  ← per D-04
- `lastAlertedAt` — timestamp (column: `last_alerted_at`), nullable  ← per D-04
- `confirmationThreshold` — integer (column: `confirmation_threshold`), not null, default 2  ← per D-05
- `createdAt` — timestamp (column: `created_at`), not null, defaultNow()
- `updatedAt` — timestamp (column: `updated_at`), not null, defaultNow()

**check_results table (per D-07, D-08):**
Columns:
- `id` — uuid primary key, defaultRandom()
- `monitorId` — uuid (column: `monitor_id`), not null, FK → monitors.id ON DELETE CASCADE
- `status` — checkStatus enum, not null
- `responseTimeMs` — integer (column: `response_time_ms`), nullable
- `errorType` — text (column: `error_type`), nullable  (values: connection_refused, dns_error, timeout, wrong_status, icmp_failed)
- `httpStatusCode` — integer (column: `http_status_code`), nullable
- `checkedAt` — timestamp (column: `checked_at`), not null, defaultNow()

Add composite index on (monitor_id, checked_at DESC):
```typescript
export const checkResultsMonitorCheckedAtIdx = index('check_results_monitor_id_checked_at_idx')
  .on(checkResultsTable.monitorId, checkResultsTable.checkedAt)
```
Note: Drizzle's `.on()` sets ASC by default; the DESC intent is documented in a comment and can be enforced via raw SQL in the migration if needed. The index must exist regardless.

**incidents table (per D-09):**
Columns:
- `id` — uuid primary key, defaultRandom()
- `monitorId` — uuid (column: `monitor_id`), not null, FK → monitors.id ON DELETE CASCADE
- `startedAt` — timestamp (column: `started_at`), not null, defaultNow()
- `resolvedAt` — timestamp (column: `resolved_at`), nullable
- `durationSeconds` — integer (column: `duration_seconds`), nullable

**notification_channels table (per D-10):**
Columns:
- `id` — uuid primary key, defaultRandom()
- `name` — text, not null
- `type` — text (column: `type`), not null  (values: 'discord', 'slack')
- `webhookUrl` — text (column: `webhook_url`), not null
- `createdAt` — timestamp (column: `created_at`), not null, defaultNow()

**monitor_notification_channels join table (per D-11):**
Columns:
- `monitorId` — uuid (column: `monitor_id`), not null, FK → monitors.id ON DELETE CASCADE
- `notificationChannelId` — uuid (column: `notification_channel_id`), not null, FK → notification_channels.id ON DELETE CASCADE
- Composite primary key on (monitorId, notificationChannelId)

**Import style:** Use `#/db/schema` alias pattern from established codebase conventions. File itself uses relative imports from drizzle-orm.
