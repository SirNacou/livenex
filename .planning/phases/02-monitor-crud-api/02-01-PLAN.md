**Endpoint specifications (per D-02 through D-16):**

**`GET /monitors`** — list all monitors
- Query: `db.select().from(monitorsTable).orderBy(desc(monitorsTable.createdAt))`
- Response: 200, array of monitor rows (per D-07)
- Import `desc` from `drizzle-orm`

**`GET /monitors/:id`** — get single monitor
- Param schema: `{ params: t.Object({ id: t.String() }) }`
- Query: select where `eq(monitorsTable.id, params.id)`
- Response: 200 with row, 404 with `{ error: "Monitor not found" }` if not found (per D-08, D-10)

**`POST /monitors`** — create monitor (per MON-01, D-05)
- Body schema — **discriminated union** (per D-11): Use `t.Union([httpBody, pingBody])` where:
  - `httpBody = t.Object({ name: t.String({ minLength: 1 }), type: t.Literal("http"), url: t.String({ format: "uri" }), interval: t.Number({ minimum: 10 }), enabled: t.Optional(t.Boolean()), showOnStatusPage: t.Optional(t.Boolean()), confirmationThreshold: t.Optional(t.Number({ minimum: 1 })) })`
  - `pingBody = t.Object({ name: t.String({ minLength: 1 }), type: t.Literal("ping"), host: t.String({ minLength: 1 }), interval: t.Number({ minimum: 10 }), enabled: t.Optional(t.Boolean()), showOnStatusPage: t.Optional(t.Boolean()), confirmationThreshold: t.Optional(t.Number({ minimum: 1 })) })`
- Insert: `db.insert(monitorsTable).values({ ...body }).returning()`
- Response: `set.status = 201`, return inserted row (per D-05)
- After insert: call `scheduler.onMonitorCreated(result.id)` (per D-14)

**`PUT /monitors/:id`** — update monitor (per MON-05, D-06)
- Param schema: `{ params: t.Object({ id: t.String() }) }`
- Body schema: `t.Object({ name: t.Optional(t.String({ minLength: 1 })), url: t.Optional(t.String({ format: "uri" })), host: t.Optional(t.String({ minLength: 1 })), interval: t.Optional(t.Number({ minimum: 10 })), enabled: t.Optional(t.Boolean()), showOnStatusPage: t.Optional(t.Boolean()), confirmationThreshold: t.Optional(t.Number({ minimum: 1 })) })`
- **Do NOT allow updating `type`, `currentStatus`, `consecutiveFailures`, `lastAlertedAt`** — those are runtime/scheduler-owned fields (per Agent's Discretion note in CONTEXT.md)
- Update: `db.update(monitorsTable).set({ ...body, updatedAt: new Date() }).where(eq(monitorsTable.id, params.id)).returning()`
- Response: 200 with updated row; 404 if no row returned (per D-06, D-10)
- After update: call `scheduler.onMonitorUpdated(updated.id)` (per D-14)

**`PATCH /monitors/:id/toggle`** — flip enabled field (per MON-06, D-06)
- Param schema: `{ params: t.Object({ id: t.String() }) }`
- No body required
- First fetch current state, then update: `db.update(monitorsTable).set({ enabled: !current.enabled, updatedAt: new Date() }).where(eq(monitorsTable.id, params.id)).returning()`
- Response: 200 with updated row; 404 if not found (per D-06, D-10)
- After toggle: call `scheduler.onMonitorToggled(updated.id)` (per D-14)

**`DELETE /monitors/:id`** — delete monitor (per MON-07, D-09)
- Param schema: `{ params: t.Object({ id: t.String() }) }`
- Delete: `db.delete(monitorsTable).where(eq(monitorsTable.id, params.id)).returning()`
- Cascade: `ON DELETE CASCADE` in schema handles `check_results` + `incidents` automatically (per D-13) — no manual cleanup needed
- Response: `set.status = 204`, no body (per D-09); 404 if row not found
- Before delete: call `scheduler.onMonitorDeleted(id)` (per D-14)

**Error pattern (per D-10):** All handlers use `set.status = 400/404/500` and return `{ error: "message" }`.

**No auth guard** — per D-16, auth is Phase 3's responsibility.
