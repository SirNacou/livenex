**Database Tables:**
1. **users** - User accounts with email, password, verification status
2. **sessions** - Session tokens with expiry and metadata (IP, user agent)
3. **api_keys** - API key storage with permissions and last-used tracking
4. **audit_logs** - Action logging for audit trail and compliance

**Key Design Patterns:**
- UUID primary keys with default generation (`defaultRandom()`)
- Automatic timestamp updates via `$onUpdate()` hooks
- Foreign key constraints with cascade delete
- Indexes on user_id (foreign key), email (unique), deleted_at (soft deletes)

**Response Envelopes:**

Success Response:
```json
{
  "ok": true,
  "data": { /* response data */ },
  "timestamp": "2026-03-27T12:00:00Z"
}
```

Error Response:
```json
{
  "ok": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable message",
    "details": { /* optional context */ }
  },
  "timestamp": "2026-03-27T12:00:00Z"
}
```

**Error Classes:**
- `ValidationError` (400) - Input validation failures
- `AuthenticationError` (401) - Missing/invalid credentials
- `AuthorizationError` (403) - Insufficient permissions
- `NotFoundError` (404) - Resource not found
- `ConflictError` (409) - Resource already exists
- `InternalError` (500) - Server errors
