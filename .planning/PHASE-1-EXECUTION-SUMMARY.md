## Common Pitfalls to Avoid

1. **Not hashing API key secrets** → Store hash only, display plaintext once
2. **Forgetting session expiry checks** → Always verify `expires_at > NOW()`
3. **Auth logic split between frontend/backend** → Auth always server-side
4. **Displaying API key multiple times** → One-time display only (D-22)
5. **Hydration mismatches** → Use TanStack Query, not imperative fetches
6. **Missing migrations** → Generate after schema, test before applying
7. **No error handling in routes** → All routes wrapped with try/catch
8. **Hardcoded secrets** → Use environment variables, generate secrets

## Help & Debugging

### If database migration fails:
```bash
# 1. Verify PostgreSQL running
docker ps | grep postgres

# 2. Check connection
PGPASSWORD=postgres psql -d livenex_dev -c "SELECT 1"

# 3. Check DATABASE_URL
cat .env.local | grep DATABASE_URL

# 4. Regenerate migration
bun run drizzle-kit generate:pg --name initial
```

### If auth routes don't compile:
```bash
# 1. Full build output
bun run build 2>&1

# 2. Type check specific file
bun run --bun tsc src/server/index.ts --noEmit

# 3. Check import paths
grep -r "from \"#/" src/server/ | head -10
```

### If tests fail:
```bash
# 1. Run with verbose output
bun run test -- --reporter=verbose

# 2. Check test database setup
# Tests may need PostgreSQL running

# 3. Review test file for mocks/fixtures needed
```
